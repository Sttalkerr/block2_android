## Практическое задание 1. Дневник

### Постановка задачи

Необходимо было реализовать приложение-дневник, в котором каждая запись сохраняется в отдельном текстовом файле во внутреннем приватном хранилище приложения.

### Реализованный функционал

- добавление новой записи;
- ввод заголовка записи;
- ввод основного текста заметки;
- автоматическая загрузка всех записей при запуске приложения;
- отображение списка записей в виде прокручиваемого списка;
- открытие записи для чтения и редактирования;
- удаление записи по долгому нажатию через контекстное меню;
- отображение пустого экрана, если записей еще нет.

### Особенности хранения данных

Для хранения записей используется директория `context.filesDir`, то есть внутреннее хранилище приложения.

Каждая запись сохраняется в отдельный `.txt` файл. Имя файла формируется по шаблону:

`timestamp_[optional_title].txt`

Примеры:

- `1712572035000.txt`
- `1712572035000_Первая_запись.txt`

Содержимое файла включает:

- заголовок записи;
- время создания;
- основной текст заметки.

### Особенности реализации по условию задания

- полное сканирование папки выполняется только один раз при запуске `ViewModel`;
- при добавлении новой записи она вручную вставляется в начало списка без повторного чтения всей папки;
- при удалении запись удаляется из текущего списка по имени файла без полного пересканирования.

### Почему выбран `context.filesDir`

Этот способ хранения подходит для личных записей, потому что:

- файлы недоступны другим приложениям;
- данные удаляются вместе с приложением;
- не нужны дополнительные разрешения;
- записи пользователя остаются приватными.

## Практическое задание 2. Фотогалерея

### Постановка задачи

Необходимо было создать личную галерею пользователя, в которой фотографии сохраняются в приватную папку приложения, а при необходимости экспортируются в общую галерею устройства.

### Реализованный функционал

- кнопка `Сделать фото`;
- запуск камеры устройства;
- сохранение фото в `getExternalFilesDir(Environment.DIRECTORY_PICTURES)`;
- вывод всех сделанных фото в `LazyVerticalGrid` с тремя колонками;
- пустой экран с предложением сделать первое фото;
- просмотр выбранного фото в диалоговом окне;
- экспорт фотографии в общую галерею;
- вывод `Snackbar` с сообщением `Фото добавлено в галерею`.

### Особенности хранения фото

Для приватного хранения фотографий используется папка:

`getExternalFilesDir(Environment.DIRECTORY_PICTURES)`

Это внешнее хранилище, но оно остается привязанным к приложению. Пользовательские фотографии:

- сохраняются в папке приложения;
- не требуют ручного управления путями общего хранилища;
- удаляются при удалении приложения.

Имя файла фотографии создается по шаблону:

`IMG_yyyyMMdd_HHmmss.jpg`

Например:

`IMG_20260408_140523.jpg`

### Разрешения

В проекте используются разрешения:

- `CAMERA` для съемки фотографии;
- `WRITE_EXTERNAL_STORAGE` только для старых версий Android (`maxSdkVersion=28`), когда это действительно необходимо для экспорта в общую папку.

### Экспорт в общую галерею

Для Android 10+ экспорт реализован через `MediaStore`, что соответствует современным требованиям Scoped Storage.

Для Android 9 и ниже предусмотрен совместимый вариант экспорта в публичную папку `Pictures` с последующим обновлением медиатеки.

## Архитектура проекта

Основные файлы проекта:

- `MainActivity.kt` — точка входа приложения;
- `Block2App.kt` — общий Compose-каркас приложения и переключение между разделами;
- `DiaryFeature.kt` — логика и интерфейс дневника;
- `GalleryFeature.kt` — логика и интерфейс фотогалереи;
- `AndroidManifest.xml` — разрешения и `FileProvider`;
- `file_paths.xml` — настройка доступа камеры к приватному файлу изображения.

## Использованные технологии

- `Jetpack Compose` для интерфейса;
- `AndroidViewModel` для хранения состояния экранов;
- `File` API для работы с текстовыми файлами дневника;
- `FileProvider` для безопасной передачи URI в камеру;
- `MediaStore` для экспорта фото в общую галерею;
- `LazyColumn` для списка записей;
- `LazyVerticalGrid` для сетки фотографий.

## Вывод

В ходе выполнения практических заданий были изучены и реализованы два сценария хранения данных в Android:

- внутреннее приватное хранение текстовых записей через `context.filesDir`;
- внешнее app-specific хранение фотографий через `getExternalFilesDir(...)` с экспортом в общую галерею через `MediaStore`.

Разработанное приложение полностью соответствует условиям заданий и демонстрирует практическое применение работы с файлами, разрешениями, камерой и современными средствами построения интерфейса на Jetpack Compose.
